# Quantum Mind

A personalized therapeutic hypnotherapy engine. Every session is uniquely generated by feeding clinical intake data through an 11-phase NLP/hypnotherapy prompt into Claude AI, then converting the script to audio via ElevenLabs TTS.

## Architecture

```
Client  →  Django REST API (JWT)
                ↓
         Celery Task Queue (Redis)
                ↓
  Claude 3.5 Sonnet → ElevenLabs TTS → DigitalOcean Spaces/S3
                ↓
           PostgreSQL (session data)
```

## Stack

| Layer | Technology |
|---|---|
| API | Django 5.0 + Django REST Framework |
| Auth | JWT (djangorestframework-simplejwt) |
| Database | PostgreSQL 15 (SQLite for local dev) |
| Task Queue | Celery 5 + Redis 7 |
| AI Script Generation | Anthropic Claude 3.5 Sonnet |
| Text-to-Speech | ElevenLabs (eleven_flash_v2_5) |
| Audio Storage | DigitalOcean Spaces / AWS S3 |
| Deployment | Docker + AWS ECS Fargate + Terraform |

## Quick Start (Local Dev)

### 1. Clone and create virtual environment

```bash
cd quantum_mind
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements/base.txt
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env — add your API keys at minimum:
# ANTHROPIC_API_KEY=sk-ant-...
# ELEVENLABS_API_KEY=...
```

### 3. Run migrations

```bash
python manage.py migrate
python manage.py createsuperuser
```

### 4. Start the development server

```bash
python manage.py runserver
```

### 5. Start Celery worker (separate terminal)

```bash
celery -A config worker --loglevel=info
```

Django admin: http://localhost:8000/admin/

## API Reference

### Authentication

```
POST /api/auth/login/       { email, password }  → { access, refresh }
POST /api/auth/refresh/     { refresh }           → { access }
POST /api/users/register/   { email, password, first_name, last_name }
GET  /api/users/profile/
```

All subsequent requests require `Authorization: Bearer <access_token>`.

### Intake Wizard

```
POST /api/intake/create/    Submit clinical intake → returns IntakeResponse with problem_category
GET  /api/intake/           List all intakes
GET  /api/intake/<id>/      Get specific intake
```

### Session Generation

```
POST /api/sessions/create/  { intake_id, duration_minutes }  → 202 Accepted
GET  /api/sessions/         List sessions (filter: ?status=completed)
GET  /api/sessions/<id>/    Poll generation status
```

**Session Status Flow:**
```
pending → generating_script → generating_audio → completed
                                               ↘ failed (credits refunded)
```

### Credits

```
GET  /api/credits/balance/         Current credit balance
GET  /api/credits/transactions/    Transaction history
GET  /api/credits/packages/        Available purchase packages
```

### Feedback & Adaptive AI

```
POST /api/feedback/mood/create/    Log mood_before (1-10) before session
PATCH /api/feedback/mood/<id>/     Update mood_after post-session
POST /api/feedback/session/        Submit effectiveness feedback → triggers profile rebuild
GET  /api/feedback/profile/        Get adaptive therapeutic profile
```

### Programs

```
GET  /api/programs/                         List available therapy programs
GET  /api/programs/<id>/                    Full program with all days
POST /api/programs/enroll/                  Enroll in a program
GET  /api/programs/my-enrollments/          User's active programs
POST /api/programs/enrollments/<id>/advance/ Mark day complete, advance to next
```

## How Session Generation Works

1. **User submits intake** (`POST /api/intake/create/`)
   - 30+ clinical fields collected across 9 wizard steps
   - Problem category auto-diagnosed from keyword scoring
   
2. **User creates session** (`POST /api/sessions/create/`)
   - Credit balance checked atomically
   - Credits deducted (free tier first, then purchased)
   - Session record created with `status=pending`
   - Celery task queued → returns `202 Accepted` immediately

3. **Celery task runs** (`sessions.generate_session`)
   - Builds personalised prompt from intake + adaptive profile + program context
   - Calls Claude 3.5 Sonnet → receives ~5,000-35,000 char script
   - Strips stage directions, splits into ≤4,500 char chunks
   - Each chunk sent to ElevenLabs TTS
   - MP3 segments concatenated and uploaded to S3/Spaces
   - Session marked `completed` with `audio_url`

4. **Client polls** (`GET /api/sessions/<id>/`)
   - Check `status` field
   - When `completed`: play `audio_url` directly from CDN

## Credit System

| Credits | Source |
|---|---|
| 10 free | Every new user |
| N purchased | Credit packages (admin-managed via Django admin) |

- 1 credit = 1 minute of session audio
- Free credits consumed first
- On session failure: full credit refund
- Full transaction ledger in `CreditTransaction` (immutable)

## Adaptive AI

After each session, users submit:
1. **Mood before/after** (1-10 scale)
2. **Session feedback** (1-5 effectiveness + which techniques resonated)

`ProfileService.rebuild_profile()` aggregates all history into a `TherapeuticProfile`:

```json
{
  "most_effective_techniques": ["Parts Integration", "Timeline Therapy"],
  "least_effective_techniques": ["Swish Pattern"],
  "average_mood_improvement_by_category": { "anxiety": 2.8, "sleep": 3.1 },
  "session_count": 14,
  "key_themes": ["work stress", "relationship", "childhood"]
}
```

This is injected into the next Claude prompt as "Returning User Context."

## Production Deployment (AWS)

### Prerequisites
- AWS CLI configured
- Terraform ≥ 1.5
- Docker + ECR access

### Deploy Infrastructure

```bash
cd terraform

# Configure variables
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars

terraform init
terraform plan
terraform apply
```

### Build and Push Docker Image

```bash
# Authenticate with ECR
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin <account>.dkr.ecr.us-east-1.amazonaws.com

# Build and push
docker build -f docker/Dockerfile -t quantum-mind .
docker tag quantum-mind:latest <account>.dkr.ecr.us-east-1.amazonaws.com/quantum-mind:latest
docker push <account>.dkr.ecr.us-east-1.amazonaws.com/quantum-mind:latest
```

### Run Migrations in Production

```bash
aws ecs run-task \
  --cluster quantum-mind-cluster \
  --task-definition quantum-mind-web \
  --overrides '{"containerOverrides":[{"name":"web","command":["python","manage.py","migrate"]}]}'
```
